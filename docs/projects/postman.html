What is covered

Auth

  • Valid token → 200/201; No token → 401
  • Invalid token → 401; Expired token → 401

GET — List / collection

  • List returns 200 and valid JSON array
  • Pagination (valid and invalid), empty list, query filters
  • Auth: 401 without token, 200 with valid token

GET — Single resource by ID

  • Valid ID → 200 and correct body, expected fields and types
  • Non-existent ID → 404; invalid ID format → 400 or 404
  • Auth: 401 without token, 200 with valid token

POST — Create

  • Valid body → 201 and Location header (or per contract)
  • Missing/invalid fields → 400 with validation message; wrong Content-Type → 415/400
  • Auth and duplicate handling (409/400) when applicable

PUT & PATCH — Update

  • Valid body + existing ID → 200 and updated resource
  • PATCH: only sent fields updated
  • 404 for non-existent ID; 400 for invalid ID or validation errors; auth checks

DELETE

  • Valid ID → 204 (or 200); GET after DELETE → 404
  • 404 for non-existent, 400/404 for invalid ID; 409 when resource in use
  • Auth: 401 without token, success with valid token

Headers, errors, performance

  • Accept: application/json and Content-Type in response
  • 4xx with error body; 5xx handled without client crash
  • Response time within configured timeout (optional)

How to run

Copy .env.example to .env and set base_url, auth_token (GoREST token). The .env file is gitignored — it never goes into the repo, so your token stays local.

Postman UI: Import REST-API-Tests.postman_collection.json and environment → select environment → Run collection.

CLI (Newman):

cd postman-api-collection
newman run collection/REST-API-Tests.postman_collection.json \
  -e environments/demo.postman_environment.json

See the repo README for up-to-date instructions.
