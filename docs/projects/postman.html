Postman API Collection

Structured Postman (or Bruno) collection for REST API testing: requests, environment variables, and automated assertions. Reusable, shareable, and runnable in CI with Newman or Bruno CLI. No secrets in repo; use env files or Postman environment for credentials.

  • Postman
  • REST API

View on GitHub

Purpose

Demonstrate API testing with Postman (or Bruno) as per industry practice. The collection includes tests for status codes, response body, and optional schema validation. Variables (e.g. base URL, API key placeholder) live in environment files; real secrets stay out of version control.

Suggested structure

postman-api-collection/
├── collection/
│   └── REST-API-Tests.postman_collection.json
├── environments/
│   ├── demo.postman_environment.json
│   └── .env.example
├── docs/
│   └── API-Scope.md
└── README.md

Collection folders can be grouped by resource or flow (e.g. Users, Posts, Auth). Each request can have pre-request scripts (e.g. token, timestamp) and a Tests tab with pm.test() and pm.expect() for assertions.

Contents (suggested)

  • Collection: Folders by resource or flow. Pre-request scripts for auth/tokens; Tests tab for status, body, optional schema validation.
  • Environments: base_url, api_key (placeholder); real values via Postman env or .env (not committed).
  • README: How to import, set environment, run collection or Newman.

How to run

Postman UI: Import collection and environment → select environment → Run collection.

CLI (Newman):

newman run collection/REST-API-Tests.postman_collection.json \
  -e environments/demo.postman_environment.json

Add your API endpoints, tests, and environment files as needed. See the repo README for up-to-date instructions.
