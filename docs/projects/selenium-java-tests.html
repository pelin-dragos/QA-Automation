Selenium Java Tests

UI automation with Selenium WebDriver, Java 17, Maven and JUnit 5. Each project is a Maven module with Page Objects, TEST_CASES.md, and TEST_RUN_CONFORMITY.md. No credentials in code; config via env and .env.example. Tests run in Firefox (WebDriverManager).

  • Selenium
  • Java 17
  • JUnit 5
  • Maven
View on GitHub

Projects overview

Six modules under selenium-java-tests/. Each has its own pom.xml, mvnw.cmd, config, base test, page objects, and tests. Run independently. Use View code to open the repo and Details for a short explanation.

What is covered

Login & Logout

  • Valid/invalid credentials, empty fields, logout from menu, redirect after logout (PROJECT_01).

Form validation

  • Email format, password strength, phone patterns, required fields, error messages (PROJECT_02).

Banking & Admin

  • Banking: login, balance, transfer, history (PROJECT_14). Admin: dashboard, list, CRUD, search (PROJECT_15).

API + UI

  • Create/update resource via RestAssured; open same URL in browser; verify JSON/data consistency (PROJECT_16).

BDD (Cucumber)

  • Gherkin features: login, logout, shopping cart, checkout, navigation, product sorting. Step definitions use Page Objects; tags for smoke/cart/checkout (PROJECT_17).

How to run

Requires Java 17+ and Firefox. Use .env.example per project for URLs and credentials (no secrets in repo).

# From a project folder (e.g. PROJECT_01)
cd selenium-java-tests/PROJECT_01_Login_Logout_Testing
.\mvnw.cmd test

# PROJECT_17 (Cucumber) — all scenarios or by tag
cd selenium-java-tests/PROJECT_17_BDD_Framework
.\mvnw.cmd test
.\mvnw.cmd test -Dcucumber.filter.tags="@smoke"

From repo root with Maven installed:

mvn test -pl selenium-java-tests/PROJECT_01_Login_Logout_Testing
